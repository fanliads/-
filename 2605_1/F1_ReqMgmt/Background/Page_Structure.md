# 页面信息结构 (Page Information Architecture)

## 1. 登录与认证

### 1.1 登录页 (Login)
- **页面类型**: 全新页面
- **核心元素**:
    - 用户名/邮箱输入框
    - 密码输入框（支持显示/隐藏切换）
    - 【登录】主按钮
    - 【忘记密码】文字链接
- **无额外导航栏**，页面居中卡片布局。

---

## 2. 核心工作区

### 全局左侧导航菜单
左侧一级菜单仅保留以下五项：
1. 看板视图（默认首页）
2. 迭代版本
3. 原始需求池
4. 产品需求池
5. 数据看板

「优先级视图」「产品经理任务」「进度看板」仅作为看板视图内的子 Tab 展示，不在一级菜单中出现。「我的待办」通过看板视图右上角用户头像下拉菜单进入。

### 2.1 看板视图 (Kanban Board) — 默认首页
- **页面类型**: 全新页面
- **顶部工具栏 (Toolbar)**:
    - **视图切换器**: [看板] | [列表] | [日历] | [优先级] | [产品经理任务] | [迭代版本] | [进度看板] — Tab 形式，当前高亮。其中「优先级」「产品经理任务」「进度看板」仅作为看板视图内的子 Tab 展示，不在左侧一级菜单中出现。
    - **筛选器组**:
        - 业务线下拉筛选（全部 / 业务线A / 业务线B...）
        - 优先级多选筛选（P0 / P1 / P2 / P3）
        - 负责人下拉筛选（支持搜索人名）
        - 迭代版本下拉筛选
    - **搜索框**: 全文搜索，支持按需求编号、标题、描述搜索
    - **操作按钮**: 【+ 新建需求】（蓝色主按钮）
- **看板主体 (Board Area)**:
    - **看板列**: 横向排列，每列代表一个需求状态。默认列：待项目经理评估 | 待产品总监判定 | 待产品组长过滤 | 待产品经理接手 | 待办需求 | 调研中 | 设计中 | 等待用户确认 | 研发中 | 测试中 | 已上线 | 已关闭
    - **列头信息**: 状态名称 + 该列需求数量角标
    - **需求卡片 (Card)**:
        - 左上角：优先级色块标签（P0红 / P1橙 / P2黄 / P3灰）
        - 标题：单行，超长截断并显示 tooltip
        - 中部：标签组（最多显示3个，超出显示 +N）
        - 底部信息行：负责人头像 + 截止日期（逾期标红） + 附件数图标 + 评论数图标
        - 右上角角标：「待我处理」红色圆点（仅当当前用户为该节点审批人时显示）
    - **拖拽交互**: 支持卡片在列之间拖拽；拖拽至目标列时，若触发审批 gate，弹出确认弹窗要求填写结论/原因；无 gate 时直接变更状态。
- **右侧快速操作**:
    - 点击卡片：打开需求详情侧滑抽屉（见 3.1）
    - 双击卡片：进入需求详情全页（见 3.2）

### 2.2 列表视图 (List View)
- **页面类型**: 全新页面
- **顶部工具栏**: 与看板视图共享相同筛选器和搜索框。
- **列表主体**:
    - **表头**: 复选框 | 需求编号 | 原始需求描述 | 当前状态 | 优先级 | 业务线 | 产品模块 | 产品经理 | 提出人 | 期望上线时间 | 提出日期 | 关联项目 | 工作量估算 | 需求单链接 | 操作
    - **行数据**: 每行展示一条需求，状态以标签形式展示。
    - **批量操作栏**: 勾选多行后，顶部浮现批量操作栏：【批量变更状态】、【批量指派负责人】、【批量关联迭代】、【批量导出】
    - **排序**: 表头点击支持升序/降序排序。

### 2.3 日历视图 (Calendar View)
- **页面类型**: 全新页面
- **顶部工具栏**: 共享筛选器，额外增加【月视图 / 周视图】切换按钮。
- **日历主体**:
    - 以日期为网格，展示截止日期的需求卡片缩略信息。
    - 支持按迭代版本高亮显示整个迭代的时间范围。
    - 点击日期格子：展示该日期的所有到期需求弹窗。

### 2.4 优先级视图 (Priority View)
- **页面类型**: 全新页面
- **设计目的**: 以优先级为首要维度组织需求，确保高优先级需求（P0/P1）获得最高视觉权重，避免重要需求被淹没。
- **顶部工具栏**: 共享筛选器（不含优先级筛选，因为本视图已按优先级分组）。
- **视图主体**:
    - **分组展示**: 页面垂直分为四个区域（P0紧急 / P1重要 / P2一般 / P3低优），每组可折叠/展开。
    - **P0 区域特殊处理**: 固定在页面顶部，背景色偏暖警示，卡片尺寸更大，始终展开不可折叠。
    - **每组内部**: 以卡片列表形式展示需求，按期望上线时间升序排列（越早截止越靠前）。
    - **卡片额外信息**: 显示「期望上线时间」「提出人」「关联项目」「工作量估算」。
    - **进度条**: 每条需求卡片底部显示细进度条（按当前状态占总流程的百分比估算）。
- **特殊交互**: P0 需求数量 > 0 时，页面顶部显示全局警示条：「当前有 N 个 P0 需求待处理，请优先关注」。

### 2.5 产品经理任务视图 (PM Task View)
- **页面类型**: 全新页面
- **设计目的**: 按产品经理维度横向对比各人的工作负载与进度，便于产品组长和管理层进行资源调配。
- **顶部工具栏**: 共享筛选器，额外增加【按人分组 / 按状态分组】切换按钮。
- **视图主体**:
    - **按人分组模式（默认）**:
        - 横向排列各产品经理的「任务列」，每列顶部显示产品经理头像、姓名、当前负责需求总数、本周到期数。
        - 每列内部按状态分色块堆叠展示需求卡片（待办 / 调研中 / 设计中 / 研发中 / 测试中）。
        - 支持拖拽需求卡片在不同产品经理之间移交（需确认弹窗）。
    - **按状态分组模式**:
        - 类似看板，但卡片按产品经理头像色环区分，便于在同一状态下快速定位负责人。
    - **负载预警**: 当某产品经理负责的需求数超过预设阈值（如10个），其列头显示橙色警示；超过紧急阈值（如15个），显示红色警示。
- **特殊交互**: 点击产品经理头像，展开侧边面板显示该 PM 的近期工作量趋势图（近4周的需求处理数）。

### 2.6 迭代版本视图 (Sprint/Release View)
- **页面类型**: 全新页面
- **设计目的**: 以迭代版本（Sprint/Release）为容器组织需求，支持从版本维度进行排期审视、进度追踪与发布规划。
- **顶部工具栏**: 共享筛选器（不含迭代筛选，因为本视图已按迭代分组），额外增加【时间线模式 / 看板模式】切换。
- **视图主体 — 时间线模式（默认）**:
    - 横向时间轴，展示各迭代版本的时间范围条（类似甘特图）。
    - 每个迭代条下方折叠/展开显示该迭代内的需求列表。
    - 迭代条颜色：规划中（灰） / 进行中（蓝） / 已延期（红） / 已发布（绿）。
    - 需求卡片在迭代时间条内按期望上线时间定位，逾期卡片自动飘红并显示逾期天数。
- **视图主体 — 看板模式**:
    - 看板列不再是「状态」，而是「迭代版本」。
    - 每列代表一个迭代，列内需求卡片按当前状态显示色标。
    - 支持需求卡片跨列拖拽，实现需求在不同迭代间的快速调整（拖出时需确认并记录原因）。
- **特殊功能**: 每个迭代列头显示完成度环形图（已完成需求数 / 总需求数）。

### 2.7 进度看板视图 (Progress Board View)
- **页面类型**: 全新页面
- **设计目的**: 弱化状态流转，强化「完成百分比」可视化，适合管理层和项目经理快速识别卡点和风险。
- **顶部工具栏**: 共享筛选器，额外增加【按需求分组 / 按业务线分组 / 按负责人分组】切换。
- **视图主体**:
    - **卡片设计**: 每张需求卡片以「大进度环」为核心视觉元素（0%-100%），环内显示完成百分比数字。
    - **进度计算规则**: 根据当前状态自动映射百分比（如：待评估0% / 设计中20% / 研发中50% / 测试中80% / 已上线100%），支持负责人手动微调。
    - **风险标识**: 进度 < 50% 且距离期望上线时间 < 3 天的需求，卡片边框飘红并显示「高风险」角标。
    - **分组展示**:
        - 按需求分组：所有需求平铺，按进度从低到高排列（优先展示卡点）。
        - 按业务线分组：业务线作为折叠区块，区块内展示该业务线下所有需求的平均进度条。
        - 按负责人分组：类似产品经理任务视图，但视觉重心在进度环而非任务数。
- **特殊交互**: 点击进度环，弹出该需求的历史进度变更记录（谁、何时、从多少改到多少）。

## 3. 需求详情与操作

### 3.1 原始需求单详情 — 侧滑抽屉 (Raw Requirement Drawer)
- **页面类型**: 全新交互组件
- **触发方式**: 在原始需求单列表/需求池中点击原始需求卡片
- **抽屉宽度**: 屏幕宽度的 50%-60%，支持拖拽调整宽度
- **标识区分**: 原始需求单卡片和详情页顶部显示「原始需求」蓝色标签，与拆分后的「产品需求」做明显视觉区分。
- **顶部固定区**:
    - 原始需求编号（可复制，格式 RAW-YYYYMMDD-NNN）
    - 当前状态标签（带颜色：待评估 / 已判定 / 已拆分 / 已拒绝 / 已挂起）
    - **审批操作栏**（仅当当前用户为该节点审批人时显示）:
        - 项目经理节点：【承接】 【拒绝】 【挂起】
        - 产品总监节点：【判定为定制化】 【判定为标准产品】 【拆分需求】
    - 【编辑】按钮（有编辑权限时显示）
    - 【关闭抽屉】X 按钮
- **内容区 — Tab 切换**:
    - **Tab 1: 概览**
        - 原始需求标题（可编辑）
        - 富文本描述区（支持 Markdown 渲染，展示商务/市场提交的原始描述）
        - **内容补充区**: 显示「补充内容」按钮，点击后展开编辑区，允许产品总监或项目经理补充技术细节、背景信息、客户诉求澄清。补充内容以「补充人 + 时间 + 内容」的形式追加展示，与原始描述区分（如灰色边框或时间轴样式）。
        - 基础信息网格：来源 / 提出人 / 提出日期 / 关联项目 / 需求单链接 / 当前处理人
        - 拆分后的产品需求列表（可点击跳转至子需求详情）
        - 附件列表（支持预览、下载、删除）
    - **Tab 2: 流转记录 (Timeline)**
        - 时间轴形式展示所有状态变更、审批结论、挂起原因、内容补充记录。
        - 每条记录显示：操作人头像 + 操作时间 + 操作类型 + 详细内容。
    - **Tab 3: 评论 (Comments)**
        - 评论流（类似聊天界面），支持 @提及、表情、附件。
        - 输入框固定在底部，支持 Ctrl+Enter 快捷发送。
- **底部操作区**:
    - 【打开全页详情】链接

### 3.2 产品需求详情 — 侧滑抽屉 (Product Requirement Drawer)
- **页面类型**: 全新交互组件
- **触发方式**: 在看板/列表/迭代视图中点击产品需求卡片
- **标识区分**: 产品需求卡片和详情页顶部显示「产品需求」绿色标签。
- **顶部固定区**:
    - 产品需求编号（可复制，格式 PRD-YYYYMMDD-NNN）
    - 当前状态标签（带颜色）
    - **审批操作栏**（仅当当前用户为该节点审批人时显示）:
        - 产品组长节点：【通过并分发】 【挂起】
        - 产品经理节点：【接手】 【开始调研】 【纳入迭代】 【提交确认】
    - 【编辑】按钮（有编辑权限时显示）
    - 【关闭抽屉】X 按钮
- **内容区 — Tab 切换**:
    - **Tab 1: 概览**
        - 需求标题（可编辑）
        - 富文本描述区（支持 Markdown 渲染）
        - 基础信息网格：优先级 / 业务线 / 需求类型（优化迭代 / 问题转需求 / 设计缺陷 / 规划功能） / 产品经理 / 迭代版本 / PRD地址 / 期望上线时间 / 工作量估算 / 价值量化评估 / 创建时间
        - 父需求（原始需求单）关联区（可点击跳转）
        - 同级子需求关联区（展示同一原始需求拆分出的其他产品需求）
        - 附件列表
    - **Tab 2: 流转记录 (Timeline)**
    - **Tab 3: 评论 (Comments)**
- **底部操作区**:
    - 【打开全页详情】链接

### 3.2 需求详情 — 全页 (Full Page)
- **页面类型**: 全新页面
- **用途**: 需要深度编辑或查看大量关联信息时使用。
- **布局**: 左侧 2/3 为详情内容区（同侧滑抽屉的 Tab 内容），右侧 1/3 为关联信息面板（关联需求、关联文档、参与人列表）。

### 3.3 新建原始需求单弹窗 (Create Raw Requirement)
- **页面类型**: 全新交互组件
- **触发方式**: 点击【+ 新建需求】按钮
- **弹窗尺寸**: 居中 Large Modal（宽度 720px）
- **适用角色**: 市场、实施、客服、商务等需求发起方
- **表单字段（精简，降低发起方填写成本）**:
    - 原始需求标题（必填，单行文本，对应「原始需求描述」）
    - 原始需求描述（富文本编辑器，鼓励粘贴客户原话或现场记录）
    - 需求来源（单选：市场 / 实施 / 客服 / 商务，自动填充当前用户所属部门）
    - 关联项目（文本输入，如「台州各区县教育局」）
    - 需求单链接（URL 输入框，关联外部文档链接）
    - 附件上传区（支持拖拽上传，多文件，如客户合同、现场照片、录音等）
- **底部按钮**: 【取消】 【提交】
- **提交后**: 原始需求单进入「待项目经理评估」状态，需求发起方可在个人中心查看自己提交的所有原始需求及其当前处理进度。

### 3.4 拆分原始需求单弹窗 (Split Raw Requirement)
- **页面类型**: 全新交互组件
- **触发方式**: 产品总监在原始需求单详情页点击【拆分需求】
- **弹窗尺寸**: 全屏或超大 Modal（宽度 960px）
- **弹窗结构**:
    - **左侧 40%**: 原始需求单信息（只读），包括原始标题、描述、附件、需求单链接，便于产品总监对照参考。
    - **右侧 60%**: 拆分操作区
        - **拆分模式选择**: 【拆分为 N 个产品需求】或【无需拆分，直接转为 1 个产品需求】
        - **产品需求列表（动态增删）**: 每个产品需求条目包含：
            - 需求标题（必填）
            - 需求描述（富文本，支持从原始描述一键复制粘贴并修改）
            - 业务线（下拉选择）
            - 产品模块（级联下拉）
            - 优先级（P0-P3）
            - 期望上线时间（日期选择器）
            - 工作量估算（人天）
            - 价值量化评估（评分+文本）
        - 【+ 新增一条产品需求】按钮
- **底部按钮**: 【取消】 【保存并拆分】
- **提交后**: 原始需求单状态变更为「已拆分」，系统自动创建对应数量的产品需求，状态均为「待产品组长过滤」，并通知对应产品组长。

### 3.5 补充内容弹窗 (Supplement Content)
- **页面类型**: 全新交互组件
- **触发方式**: 在原始需求单详情页点击【补充内容】
- **弹窗尺寸**: 居中 Medium Modal（宽度 560px）
- **适用场景**: 商务/市场提交的原始描述过于粗糙或缺少关键信息，由项目经理或产品总监进行内容补充和澄清。
- **表单字段**:
    - 补充类型（单选：客户背景补充 / 技术细节补充 / 需求范围澄清 / 其他）
    - 补充内容（富文本编辑器，必填）
    - 关联附件（可选上传）
- **底部按钮**: 【取消】 【提交补充】
- **提交后**: 补充内容以时间轴形式追加展示在原始需求单的「概览」Tab 中，与原始描述视觉区分，并记录补充人和补充时间。同时发送通知给需求提出人，告知其原始需求已被补充内容。

## 4. 个人与管理工作区

### 4.1 我的待办 (My Tasks)
- **页面类型**: 全新页面
- **用途**: 各角色登录后快速查看需要自己处理的需求。
- **布局**: 卡片式列表，按「待我处理」和「已处理（本周）」分组。
- **卡片内容**: 同看板视图卡片，额外显示「还需 X 小时截止」倒计时。
- **快捷操作**: 卡片上直接显示对应节点的审批按钮（如【承接】【判定】等），无需打开详情即可一键处理。

### 4.2 原始需求池 (Raw Requirement Pool)
- **页面类型**: 全新页面
- **用途**: 产品总监和项目经理查看所有原始需求单，进行承接评估、范围判定和内容补充。
- **布局**: 统一平铺列表视图（不再按状态分组），支持水平滚动查看全部字段。
- **表头字段（18列）**:
    | 字段名 | 说明 |
    |---------|------|
    | 业务线 | 下拉选择，可编辑 |
    | 原始需求标题 | 锁定列，始终显示 |
    | 需求单 | 链接形式，点击跳转需求单详情 |
    | 需求备注 | 简短文字备注，可编辑 |
    | 登记人 | 自动取录入人员姓名，只读 |
    | 提出人 | 支持编辑 |
    | 提出时间 | 日期时间，可编辑 |
    | 当前状态 | 枚举值：空 / 评估中 / 待设计 / 已设计 / 开发中 / 已上线 / 拒绝 / 挂起 / 验收不通过 |
    | 产品需求定义 | 文字描述，可编辑 |
    | 产品经理 | 下拉选择，可编辑 |
    | 项目经理 | 下拉选择，可编辑 |
    | 优先级定级 | P0/P1/P2/P3，支持AI自动定级 + 人工调整 |
    | 是否加急插队 | 是/否，可编辑 |
    | 加急原因 | 当“是否加急插队”为“是”时必填 |
    | 关联项目 | 文本输入，可编辑 |
    | 期望上线时间 | 日期选择，可编辑 |
    | 价值量化评估 | 评分+评价数，可编辑 |
    | 关联产品需求池 | 链接列表，一个原始需求可能关联多条产品需求 |
- **编辑记录**: 所有字段的编辑操作均以 Timeline 形式记录并展示，包括操作人、操作时间、字段名、旧值、新值。点击每行的“编辑记录”按钮可查看。
- **AI 优先级定级**:
    - 系统内置一段文字化的优先级判定规则（描述 P0/P1/P2/P3 的判定标准）。
    - 需求创建时，系统调用 AI 接口，将需求内容 + 判定规则发送给 AI，自动返回优先级结果及判定理由。
    - AI 定级结果以紫色「AI」徽章标记，与人工定级区分。
    - 用户可在 AI 定级弹窗中手动调整优先级，需填写调整原因，调整后标记为「人工」。
    - 所有定级操作（AI/人工）均记入编辑记录 Timeline。
- **自定义表头设置**:
    - 工具栏右侧提供「⚙ 自定义表头设置」入口。
    - 弹窗内以拖拽列表形式展示所有 18 列，每列带开关控制显示/隐藏。
    - 「原始需求标题」为锁定列，始终显示，不可关闭。
    - 用户拖拽可调整列顺序，设置保存后对当前用户生效。
    - 工具栏显示当前「显示 N/18 列」的计数提示。
- **拆分需求**:
    - 每行操作列提供「✎ 拆分」按钮，点击弹出拆分弹窗。
    - 拆分意味着将一条原始需求拆分为多条产品需求，转入产品需求池。
    - **两种拆分模式**：
        - **AI 拆分**：点击「开始 AI 拆分」按钮，AI 自动分析原始需求文本，按条目式拆分为 1、2、3... 多条独立产品需求。每条需求包含文字描述，部分条目可能带图片/原型图。AI 拆分结果可人工二次编辑后确认。
        - **手工拆分**：手动添加/删除/插入产品需求条目。每条需求包含：文字描述、截图/原型图上传（支持多图）、产品经理分配、业务线选择。
    - 弹窗左侧展示原始需求信息（标题、备注、业务线等），右侧为拆分操作区。
    - 确认后自动生成产品需求编号并转入产品需求池，同时更新原始需求的「关联产品需求池」字段。
    - 所有拆分操作记入编辑记录 Timeline。

### 4.3 产品需求池 (Product Requirement Pool)
- **页面类型**: 全新页面
- **用途**: 产品组长查看待过滤和已过滤的产品需求，进行分发操作；同时支持直接创建产品需求。
- **布局**: 列表视图为主，按状态分组（待指派 | 待产品组长过滤 | 已分发 | 已挂起）。
- **表头字段（10列）**: 产品需求编号 | 原始需求标题 | 优先级 | 业务线 | 原始需求链接 | 需求类型 | PRD地址 | 迭代版本 | 创建人 | 处理人
- **字段说明**:
    | 字段名 | 说明 |
    |---------|------|
    | 产品需求编号 | 系统自动生成，格式 PRD-YYYYMMDD-NNN，只读标识列 |
    | 原始需求标题 | 关联的原始需求单标题，可编辑；直接创建的需求无原始需求，显示「—」 |
    | 优先级 | P0/P1/P2/P3，下拉选择，可编辑 |
    | 业务线 | 下拉选择，可编辑 |
    | 原始需求链接 | 链接形式，点击在当前页面打开原始需求预览抽屉（右侧滑出），展示原始需求全部信息（标题、状态、优先级、业务线、产品经理、项目经理、加急、备注、产品需求定义、关联产品需求等），不跳转离开当前页面；直接创建的需求显示「—」 |
    | 需求类型 | 下拉选择：优化迭代 / 问题转需求 / 设计缺陷 / 规划功能，可编辑 |
    | PRD地址 | URL 输入框，关联 PRD 文档链接，可编辑 |
    | 迭代版本 | 下拉选择关联迭代版本，可编辑 |
    | 创建人 | 自动取当前登录人姓名，只读 |
    | 处理人 | 下拉选择产品经理，可编辑 |
- **所有字段均支持编辑**（创建人只读），编辑记录以 Timeline 形式记录并可展示。
- **直接创建产品需求**:
    - 工具栏提供【+ 新建产品需求】按钮，点击弹出创建弹窗。
    - 弹窗表单字段：需求标题（必填）、优先级、业务线、需求类型、PRD地址、迭代版本、处理人。
    - **创建人自动取当前登录人**，不可修改。
    - 直接创建的需求无原始需求关联，「原始需求标题」和「原始需求链接」字段显示「—」。
    - 在分组视图中，直接创建的需求作为独立行展示（无原始需求分组头），以「直接创建」绿色微标签区别于拆分需求。
- **指派与转发机制**:
    - **创建人 = 处理人**：创建后直接进入「待产品组长过滤」状态，产品组长可正常分发。
    - **创建人 ≠ 处理人**：创建后进入「待指派」状态，需产品组长或产品总监确认转发并指派任务。产品组长/产品总监可在行内重新指派处理人，确认后状态变更为「待产品组长过滤」。
- **双展示模式**: 工具栏提供视图切换按钮，支持两种列表展示方式：
    - **按原始需求分组（默认）**: 以一个原始需求为父级分组头，下方展开其拆分出的 N 个产品需求。分组头行显示原始需求编号、原始需求标题、拆分数量徽标（如「拆分 3 条」）及折叠/展开箭头；子行缩进展示各产品需求的完整字段信息。直接创建的需求无分组头，作为独立行展示。支持点击父级行折叠/展开子需求。
    - **打散展示**: 每条产品需求作为独立条目平铺展示，不显示分组关系，「原始需求标题」列体现其归属（直接创建的显示「—」）。适合快速扫描全部产品需求。
- **多维度筛选**:
    - 工具栏下方提供筛选栏，支持按以下字段独立筛选：优先级、业务线、需求类型、迭代版本（含「未关联」选项）、创建人、处理人。
    - 每个筛选器均为下拉单选，默认「全部」；激活时筛选器高亮显示，右侧显示「显示 N / 总 M 条」计数。
    - 支持多筛选条件联合生效（AND 逻辑）。
    - 搜索框同步接入筛选逻辑，支持按需求编号、需求标题实时搜索。
    - 提供「清除筛选」按钮一键重置所有筛选条件。
- **特殊功能**: 产品组长可在列表行内直接选择业务线产品经理进行快速分发，分发后状态自动变更为「待产品经理接手」。

### 4.3 迭代管理 (Sprint Management)
- **页面类型**: 全新页面
- **用途**: 管理迭代版本，查看迭代内的需求排期与进度。
- **布局**:
    - 左侧：迭代列表（显示迭代名称、时间范围、状态：规划中 / 进行中 / 已结束）
    - 右侧：选中迭代的详情，包括迭代看板、需求列表、燃尽图（可选）
    - 顶部：【新建迭代】按钮，弹出表单（名称 / 开始日期 / 结束日期 / 目标描述）

## 5. 数据分析

### 5.1 数据看板 (Dashboard)
- **页面类型**: 全新页面
- **用途**: 管理层查看需求大盘与价值回归数据。
- **核心模块**:
    - **关键指标卡**: 本月新增需求数 | 本月完成需求数 | 平均需求处理周期 | 挂起率
    - **需求状态分布**: 饼图 / 环形图
    - **各业务线吞吐量**: 柱状图（横轴：业务线，纵轴：需求数）
    - **迭代燃尽/燃起图**: 折线图（可选，用于追踪迭代进度）
    - **价值回归模块**: 已评估需求的平均评分排行、高价值需求标签云
- **筛选器**: 时间范围（本周/本月/本季度/自定义）、业务线多选

## 6. 系统设置

### 6.1 状态流配置 (Status Flow Settings)
- **页面类型**: 全新页面
- **用途**: 管理员自定义需求状态及流转规则。
- **核心功能**:
    - 看板列增删改（状态名称、颜色、顺序）
    - 流转规则设置：哪些角色可以将需求从状态A流转到状态B
    - 审批 gate 配置：在哪些状态变更时需要强制填写原因或等待特定角色确认

### 6.2 用户与权限 (User & Permissions)
- **页面类型**: 全新页面
- **核心功能**:
    - 用户列表（支持邀请、禁用、重置密码）
    - 角色管理（预设五角色 + 自定义角色）
    - 权限矩阵：按角色配置页面访问权限、操作权限（如是否能删除需求、是否能修改他人创建的需求）

### 6.3 业务线与标签 (Business Lines & Tags)
- **页面类型**: 全新页面
- **核心功能**: 业务线增删改、标签增删改、标签颜色配置。
