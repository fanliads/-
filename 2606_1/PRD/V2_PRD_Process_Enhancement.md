# V2 PRD - 流程增强版

## 1. 目标

在 V1 主链路稳定基础上，完善需求流转效率和迭代流程自动化能力。

## 2. 范围

### In Scope
- 原始需求看板；
- 产品需求看板；
- 双看板统一筛选框架；
- 迭代详情中的真实进度看板；
- 团队默认迭代周期配置；
- 自动创建下一期迭代；
- 原始需求与产品需求流程视图统一。
- 原始需求拆分后的持续新增、编辑、作废子产品需求；
- 页面级权限治理。

### Out of Scope
- AI 优先级；
- 外部身份集成；
- 外部提交页；
- 价值评估模型。

## 3. 核心需求

### 3.1 双看板
- 看板支持切换“原始需求 / 产品需求”；
- 不同类型看板展示不同状态列；
- 共用优先级、业务线、负责人筛选；
- 产品需求看板保留迭代筛选。

### 3.2 自动创建迭代
- 以团队为粒度维护默认周期；
- 到达周期边界时自动创建下一期迭代；
- 若同团队同周期同时间窗已存在迭代，则跳过；
- 自动创建记录应可追溯。

### 3.3 持续拆分维护
- 已拆分原始需求下允许继续新增子产品需求；
- 子产品需求支持继续编辑；
- 子产品需求支持业务作废并保留关系与日志；
- 原始需求详情继续展示全部关联子产品需求。

### 3.4 页面级权限治理
- 未授权角色仅可查看，不展示编辑入口；
- 原始需求池、产品需求池、迭代管理分别按页面能力收口；
- 看板拖拽与迭代纳入/移出需求按权限控制。

## 4. 验收标准

- 原始需求和产品需求都能在看板展示；
- 自动创建不会重复生成迭代；
- 团队默认周期可生效；
- 迭代详情进度视图与真实状态一致。
