# 发布/回滚执行清单模板

## 使用说明

- 发布前复制一份作为当次执行记录
- 每完成一项就勾选
- 如果发生回滚，也在同一份记录中补充
- 建议将填写后的记录保存到团队发布记录中

---

## 一、发布基本信息

```text
项目名称：
发布环境：
发布版本：
上一个稳定版本：
发布时间：
发布执行人：
验收人：
是否涉及数据库变更：
是否涉及配置变更：
备注：
```

---

## 二、发布前检查清单

### 代码与版本

- [ ] 本次发布代码已合并到目标分支
- [ ] 本次发布代码已 push 到远程仓库
- [ ] 本次发布 tag 已创建
- [ ] tag 名称已确认无误
- [ ] 已确认本次发布内容

### 配置与依赖

- [ ] 生产环境 `.env.production` 已确认
- [ ] 数据库连接配置已确认
- [ ] `JWT_SECRET` 已确认
- [ ] 第三方接口配置已确认
- [ ] Docker / Java / MySQL / Redis 状态正常

### 数据与风险

- [ ] 已确认是否包含数据库脚本
- [ ] 如有需要，数据库已备份
- [ ] 已确认是否需要灰度或错峰发布
- [ ] 已确认回滚版本

---

## 三、发布执行清单

- [ ] 已登录目标服务器
- [ ] 已进入部署目录
- [ ] 已拉取远程 tag
- [ ] 已切换到目标版本
- [ ] 已执行构建
- [ ] 已执行启动或重启
- [ ] 已完成服务状态检查
- [ ] 已完成日志检查

### 本次执行命令记录

```bash
git fetch --all --tags
git checkout tags/<version>
bash deploy/deploy.sh deploy tag <version>
```

### 实际执行记录

```text
命令执行结果：


异常信息：


处理动作：
```

---

## 四、发布后验收清单

### 技术验收

- [ ] 服务已正常启动
- [ ] 健康检查通过
- [ ] 日志中无明显致命错误
- [ ] 关键接口响应正常

### 业务验收

- [ ] 登录功能正常
- [ ] 首页或核心页面可访问
- [ ] 本次改动相关页面可访问
- [ ] 本次改动相关核心流程正常
- [ ] 无明显阻断性问题

### 发布结果

- [ ] 发布成功
- [ ] 进入观察期
- [ ] 需要回滚

---

## 五、回滚触发判断

满足任一情况，优先评估回滚：

- [ ] 无法登录
- [ ] 无法提交核心业务数据
- [ ] 核心页面大面积报错
- [ ] 服务启动失败
- [ ] 大量 500 错误
- [ ] 数据异常且影响业务

---

## 六、回滚执行清单

```text
回滚版本：
回滚开始时间：
回滚执行人：
回滚原因：
```

- [ ] 已确认回滚版本
- [ ] 已通知相关人员
- [ ] 已执行回滚命令
- [ ] 已完成服务恢复检查
- [ ] 已完成核心流程验证
- [ ] 已记录故障现象和日志

### 回滚命令模板

```bash
git fetch --all --tags
git checkout tags/<rollback-version>
bash deploy/deploy.sh rollback <rollback-version>
```

### 回滚结果记录

```text

回滚后验证结果：
```
