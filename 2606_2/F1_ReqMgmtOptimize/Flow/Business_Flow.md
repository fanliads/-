# 业务流程图

## 流程说明

本次流程围绕“规则配置 -> 原始需求提交 -> AI 自动判级 -> 人工复核/修正 -> 规则持续优化”展开。核心原则如下：

1. 优先级始终由同一个字段承载，不拆分 AI 建议值和人工最终值。
2. AI 自动判级是默认入口，但人工可以覆写。
3. AI 判级依赖当前生效规则版本和固定提示词模板。
4. 每次 AI 判级或人工改写都要留下可追溯记录。

```mermaid
flowchart TD
    subgraph Admin["管理员 / 产品负责人"]
        A1["编辑优先级规则文本"]
        A2["保存并发布规则版本"]
    end

    subgraph Requester["需求发起人"]
        R1["提交原始需求"]
    end

    subgraph System["系统 / AI判级服务"]
        S1["创建原始需求记录"]
        S2["读取当前生效规则版本"]
        S3["拼装提示词模板 + 需求上下文"]
        S4{"AI调用是否成功且结果合法"}
        S5["写入 priority 字段"]
        S6["记录判级摘要 / 规则版本 / 判定时间 / 来源"]
        S7["按降级策略处理"]
    end

    subgraph Reviewer["项目经理 / 产品负责人"]
        P1["查看当前优先级与判级摘要"]
        P2{"是否认可当前优先级"}
        P3["人工修改 priority 字段"]
        P4["记录人工修改日志"]
        P5{"是否需要重新 AI 判级"}
    end

    subgraph Governance["规则优化闭环"]
        G1["复盘误判案例"]
        G2["调整规则文本"]
    end

    A1 --> A2
    A2 --> S2

    R1 --> S1
    S1 --> S2
    S2 --> S3
    S3 --> S4

    S4 -->|是| S5
    S5 --> S6
    S6 --> P1

    S4 -->|否| S7
    S7 --> P1

    P1 --> P2
    P2 -->|是| P5
    P2 -->|否| P3
    P3 --> P4
    P4 --> P5

    P5 -->|需要| S2
    P5 -->|不需要| G1

    G1 --> G2
    G2 --> A1
```
